## Why Rime

<CardGrid stagger>
  <Card title="Functions, not jobs" icon="approve-check">
    A node is a function over dataframes — not a task that wires I/O. You write what each step computes; the runtime owns reading, writing, serialization, and language boundaries. The dbt mental model, extended past SQL.
  </Card>
  <Card title="One DAG, four languages" icon="puzzle">
    SQL for joins, Python for ML, R for stats, JavaScript for everything else. Same pipeline, named slots, typed boundaries. Dataframes cross language borders over Arrow IPC.
  </Card>
  <Card title="Reproducible by default" icon="seti:lock">
    Content-addressed caching, deterministic outputs, freeze-able snapshots. Same script + same inputs = same artifact, every time. No "works on my machine."
  </Card>
  <Card title="Publishable narratives" icon="document">
    Pair your DAG with a `report.yaml` and render a publishable HTML report. Tables, stats, prose — one render step, one document, one source of truth.
  </Card>
  <Card title="Visual editor (preview)" icon="laptop">
    Rime Editor is a desktop app for authoring pipelines visually. Top-down DAG canvas, integrated Monaco code editor, live validation. Closed beta; download coming soon.
  </Card>
</CardGrid>
