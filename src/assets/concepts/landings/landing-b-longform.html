RIME · v0.1 · Apache 2.0

Nodes are functions over dataframes.

Not jobs that wire I/O. Not tasks that read from S3 and write to S3. In Rime, you write what each step computes — in SQL, Python, R, or JavaScript — and the runtime owns the rest: reading, writing, serialization, language boundaries, caching.

SQL Python R JavaScript

Researchers and data journalists shouldn't have to write boilerplate just to move a dataframe between two steps of an analysis. Rime treats each node as a function over dataframes — you describe what it computes, and the runtime owns I/O, serialization, language boundaries, and caching. The dbt mental model, extended past SQL into the rest of the data stack.

The runtime owns the boilerplate.

A Rime project is a single pipeline.dag.yaml. Each node names a function and declares which dataframes feed it. There is no read_csv() at the top of a script and no to_parquet() at the bottom — Rime reads each input as a pandas DataFrame, R tibble, DuckDB table, or Arrow table; runs your function; captures the return value; hashes it into a content address; and hands it to the next node. Same protocol across all four languages, no glue code between them.

# pipeline.dag.yaml
version: '2.1'
nodes:
  cars:
    kind: sql_source
    source: queries/cars.sql
  embed:
    kind: python_script
    source: scripts/umap.py
    in: { table: cars }
  trends:
    kind: r_script
    source: scripts/efficiency.R
    in: { features: embed }
  test:
    kind: t_test
    in: { x: cars.us_mpg, y: cars.japan_mpg }

Built different from Hex and Deepnote.

Hex and Deepnote both wrap notebooks in modern UIs with reactive cells — fine products for exploration. Rime is different on three axes. Nodes are functions with typed signatures, not cells reaching into a notebook's global scope, so there is no hidden state across steps. The DAG is a first-class declarative artifact — a pipeline.dag.yaml you can diff, commit, and code-review — not implicit cell order or hidden in metadata. And the whole stack is Apache 2.0: runs on your laptop, in your CI, in your cluster. No SaaS lock-in. No per-seat licensing. No telemetry.

"You write the function. The runtime does the rest."

Built different from Airflow.

Airflow, Prefect, and Dagster treat each step as a task that does its own I/O: read from S3, transform, write back. Coordination happens via storage paths — the side effect is the contract. Rime inverts this: side effects are the runtime's job, functions just compute. Dataframes coordinate via typed ports. Reproducibility is a property of the model, not extra work you have to do.

Built different from Snakemake.

If you do bioinformatics on Snakemake or Nextflow, keep doing that — they're mature and you have nf-core's library of recipes. Rime is for a different audience: data journalists, social scientists, researchers shipping a study or a blog post. Dataframes instead of file paths. YAML instead of a Python DSL. A visual editor instead of vim. Statistical nodes built in.
